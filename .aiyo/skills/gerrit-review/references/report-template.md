# 代码审查报告模板

## 标准审查报告结构

```markdown
# 代码审查报告 - Change {change_number}

## 基本信息
- **提交标题**: {subject}
- **作者**: {owner}
- **项目**: {project}
- **分支**: {branch}
- **状态**: {status}
- **Patch Set**: {patch_set}

## 变更摘要
- **修改文件数**: {file_count}
- **新增行数**: {insertions}
- **删除行数**: {deletions}

## 文件变更详情
{file_list_with_diff}

## 代码审查分析

### 1. 代码质量
- [ ] 代码风格一致性
- [ ] 命名规范
- [ ] 注释完整性
- [ ] 错误处理

### 2. 逻辑正确性
- [ ] 算法正确性
- [ ] 边界条件处理
- [ ] 并发安全性
- [ ] 资源泄漏检查

### 3. 架构设计
- [ ] 模块耦合度
- [ ] 接口兼容性
- [ ] 可扩展性
- [ ] 性能影响

### 4. 测试覆盖
- [ ] 单元测试
- [ ] 集成测试
- [ ] 边界测试

## 审查结论
**状态**: [✅ 通过 / ⚠️ 有条件通过 / ❌ 需要修改]

**评分**: {code_review_vote} (Code-Review)

**主要问题**:
1. {issue_1}
2. {issue_2}

**建议**:
1. {suggestion_1}
2. {suggestion_2}
```

## 审查检查清单

### 通用检查项
- [ ] 提交信息是否清晰描述变更目的
- [ ] 变更范围是否聚焦（一个变更只做一件事）
- [ ] 是否有无关的代码变更（如调试代码、格式混乱）
- [ ] 版权头是否完整（如适用）

### C/C++ 特定检查项
- [ ] 内存管理（malloc/free 配对，智能指针使用）
- [ ] 缓冲区溢出风险
- [ ] 空指针检查
- [ ] 整数溢出
- [ ] 线程安全

### Python 特定检查项
- [ ] 异常处理
- [ ] 类型注解
- [ ] 导入排序
- [ ] docstring 完整性

### 安全相关检查项
- [ ] 敏感信息泄露（密码、密钥）
- [ ] 输入验证
- [ ] SQL 注入风险
- [ ] 命令注入风险
